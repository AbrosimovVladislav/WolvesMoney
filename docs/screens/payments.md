# Экран: Payments

**Компонент:** `src/components/PaymentsView.tsx`
**Открывается из:** экрана Ice Time при нажатии на карточку тренировки

---

## Назначение

Ввод и сохранение платежей от каждого игрока за конкретную тренировку. Ключевой рабочий экран — используется после каждой тренировки.

---

## Навигация

- Нижняя панель навигации **скрыта** на этом экране
- Кнопка **[← Back]** возвращает на экран Ice Time

---

## Структура экрана

### 1. Заголовок
- Кнопка Back (←)
- "PAYMENTS" + дата тренировки

### 2. Финансовая сводка тренировки
Карточка с тремя показателями:
| Показатель | Значение | Цвет |
|-----------|---------|------|
| Collected | текущая сумма введённых платежей | зелёный |
| Ice Cost | стоимость льда тренировки | красный |
| Result | Collected − Ice Cost | зелёный/красный |

Прогресс-бар: заполненность = `collected / iceCost * 100%`
Подпись: "X% of ice cost covered"

### 3. Быстрые действия (2 кнопки)
- **Fill All (1 500)** — установить всем игрокам стандартный взнос 1 500 RSD
- **Clear All** — обнулить всех игроков

### 4. Список игроков с инпутами
Для каждого игрока:
- **Аватар** с инициалами
- **Имя** игрока
- **Текущий баланс** игрока (мелкий подзаголовок):
  - Зелёный: "Credit: +N RSD"
  - Красный: "Debt: −N RSD"
  - Скрыт если balance = 0
- **Быстрые кнопки:**
  - **[✕]** — поставить 0 (подсвечивается красным если выбрано)
  - **[1.5k]** — поставить 1 500 RSD (подсвечивается синим если выбрано)
- **Числовой инпут** (72px) — произвольная сумма
  - Фон зелёный если amount > 0
  - Серый если amount = 0

### 5. Кнопка "Save All Payments"
- До сохранения: "💾 Save All Payments"
- После сохранения: "✓ Saved!" (кнопка остаётся, статус меняется)
- Toast: "Payments saved!"

---

## Логика сохранения

При нажатии Save:
1. Старые платежи для этой тренировки удаляются
2. Откатываются старые изменения балансов игроков
3. Вставляются новые записи платежей для всех игроков (даже с `amount = 0`)
4. Пересчитываются балансы игроков: `diff = amount - 1500`
5. Обновляются `total_collected` и `result_balance` тренировки
6. Обновляется `team_balance` на разницу нового и старого результата

---

## Бизнес-правила

- Стандартный взнос: **1 500 RSD** (`DEFAULT_PLAYER_FEE`)
- Сохранение полностью **перезаписывает** платежи тренировки (upsert через delete + insert)
- Игроки с `amount = 0` всё равно записываются в БД
- Баланс игрока меняется только на отклонение от стандарта:
  - 2 000 RSD → balance +500
  - 1 000 RSD → balance −500
  - 1 500 RSD → balance не меняется
  - 0 RSD → balance −1 500
- Можно сохранять несколько раз — каждый раз пересчитывается корректно
