# Экран: Payments

**Компонент:** `src/components/PaymentsView.tsx`
**Открывается из:** экрана Ice Time при нажатии на карточку тренировки

---

## Назначение

Ввод посещаемости и платежей от каждого игрока за конкретную тренировку. Ключевой рабочий экран — используется после каждой тренировки.

---

## Навигация

- Нижняя панель навигации **скрыта** на этом экране
- Кнопка **[← Back]** возвращает на экран Ice Time

---

## Структура экрана

### 1. Заголовок
- Кнопка Back (←)
- "PAYMENTS" + дата тренировки

### 2. Финансовая сводка тренировки
Карточка с тремя показателями:
| Показатель | Значение | Цвет |
|-----------|---------|------|
| Collected | текущая сумма введённых платежей | зелёный |
| Ice Cost | стоимость льда тренировки | красный |
| Result | Collected − Ice Cost | зелёный/красный |

Прогресс-бар: заполненность = `collected / iceCost * 100%`
Подпись: "X% of ice cost covered"

### 3. Быстрые действия (2 кнопки)
- **Fill All** — отметить всех как пришедших и выставить каждому его персональный `default_fee`
- **Clear All** — снять отметку посещаемости у всех (все = "не пришёл")

### 4. Список игроков

Для каждого игрока отображается строка в два шага:

**Шаг 1 — Посещаемость (всегда видна):**
- Аватар с инициалами
- Имя игрока + текущий баланс (Credit/Debt, если ≠ 0)
- Кнопка-тоггл посещаемости:
  - **[Не пришёл]** — серый/красный фон, активен по умолчанию
  - **[Пришёл]** — зелёный фон, при нажатии переключает в режим оплаты

**Шаг 2 — Сумма платежа (видна только если `attended = true`):**
- Быстрая кнопка **[✕]** — выставить 0 (подсвечивается красным если выбрано)
- Быстрая кнопка **[N RSD]** — выставить персональный `player.default_fee` (подсвечивается синим если выбрано)
- Числовой инпут (72px) — произвольная сумма
  - Зелёный фон если amount > 0
  - Серый фон если amount = 0

**Визуальный пример строки:**
```
[АВ] Андрей Волков               [  Пришёл  ]
     Debt: -500 RSD    [✕] [1.5k] [  1500   ]

[ПС] Пётр Сидоров                [ Не пришёл ]
```

### 5. Кнопка "Save All Payments"
- До сохранения: "💾 Save All Payments"
- После сохранения: "✓ Saved!" (кнопка остаётся, статус меняется)
- Toast: "Payments saved!"

---

## Логика сохранения

При нажатии Save:
1. Загружаются старые payment-записи для этой тренировки
2. Откатываются старые изменения балансов игроков (только где `attended = true`)
3. Удаляются все старые payment-записи для тренировки
4. Вставляются новые записи для всех игроков (с полем `attended` и `amount`)
5. Пересчитываются балансы игроков по новой логике (см. ниже)
6. Обновляются `total_collected` и `result_balance` тренировки
7. Обновляется `team_balance` на разницу нового и старого результата

---

## Бизнес-правила

### Логика баланса игрока

| attended | amount | Изменение balance |
|----------|--------|-------------------|
| false | 0 | **не меняется** — не пришёл, ничего не должен |
| true | 0 | `−player.default_fee` — пришёл, не заплатил → должен |
| true | N > 0 | `+(N − player.default_fee)` — разница от персонального стандарта |

### Прочее
- **Fill All** выставляет каждому игроку его персональный `player.default_fee`, не глобальный
- Можно сохранять несколько раз — каждый раз пересчитывается корректно (idempotent)
- Игроки с `attended = false` сохраняются в БД с `amount = 0, attended = false` — это важно для корректного отката при повторном сохранении
- В `total_collected` тренировки учитываются только суммы от пришедших игроков
