# Экран: Players

**Компонент:** `src/components/Players.tsx`
**Вкладка:** Players (вторая)

---

## Назначение

Управление составом команды. Позволяет добавлять и удалять игроков, смотреть их баланс и историю платежей.

---

## Структура экрана

### 1. Заголовок + кнопка "Add"
- Заголовок "PLAYERS"
- Кнопка **[+ Add]** — открывает модальное окно добавления

### 2. Поиск
- Инпут с плейсхолдером "🔍 Search players..."
- Фильтрует список в реальном времени по имени (case-insensitive)

### 3. Счётчики (3 карточки в ряд)
| Карточка | Значение | Цвет |
|----------|---------|------|
| Total | общее кол-во игроков | синий |
| Debtors | кол-во игроков с balance < 0 | красный |
| Credits | кол-во игроков с balance > 0 | зелёный |

### 4. Список игроков
Каждая строка содержит:
- **Аватар** с инициалами (цвет фона зависит от баланса):
  - Красный градиент → долг (balance < 0)
  - Зелёный градиент → кредит (balance > 0)
  - Синий градиент → ровно (balance = 0)
- **Имя** игрока
- **Кол-во платежей** + взнос если отличается от стандарта: "Fee: N RSD" (мелкий подзаголовок)
- **Тег баланса** (зелёный `+`, красный `−`, серый `0`)

Нажатие на строку → открывает **детальное модальное окно** игрока.

---

## Модальное окно: Add Player

Поля:
- **Full Name** — текстовый инпут (автофокус)
- **Default Fee (RSD)** — числовой инпут, дефолт: 1 500
- Enter — подтверждение добавления

Кнопки: **Cancel** / **Add Player**

После добавления:
- Игрок создаётся с balance = 0 и указанным default_fee
- Показывается Toast: "Player added"

---

## Модальное окно: Player Detail

Содержит:
- Аватар с инициалами (52×52px)
- Имя игрока (uppercase)
- Тег статуса: **Credit: +N RSD** / **Debt: −N RSD** / **Even: 0 RSD**
- Секция **Default Fee**:
  - Числовой инпут с текущим значением `default_fee`
  - Кнопка **[Save Fee]** — сохраняет новое значение
  - Toast: "Fee updated"
- Список "Payment History" — все платежи этого игрока, отсортированные от новых к старым:
  - Дата тренировки (левый столбец)
  - Статус посещения: "Attended" (зелёный) / "Absent" (серый)
  - Сумма платежа в RSD (правый столбец, зелёный; 0 если не платил)
  - Если платежей нет: "No payments yet"
- Кнопка **[🗑 Remove Player]** — удаляет игрока

После удаления:
- Модальное окно закрывается
- Toast: "Player removed"

---

## Бизнес-правила

- Игрок создаётся с `balance = 0` и `default_fee = 1500`
- Баланс изменяется только через сохранение платежей (экран Payments)
- `default_fee` используется в экране Payments как стандартный взнос для этого игрока
- Изменение `default_fee` **не пересчитывает** прошлые балансы — действует только на будущие тренировки
- Удаление игрока **не пересчитывает** командный баланс (только удаляет запись)
